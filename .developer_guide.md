## 🚀 When You Update Code - Complete Workflow

### 1. Code Development & Testing

```bash
# 1. Make your changes in R/, src/, etc.
# 2. Test locally with pixi
pixi run R -e "devtools::load_all(); # test your changes"

# 3. Run regression tests to ensure nothing broke
./testing/run_regression_test.sh comprehensive

# 4. If tests pass, continue to release
```

### 2. Version Update & Automated Release

```bash
# ONE COMMAND DOES EVERYTHING:
./scripts/update_version.sh 0.3.4

# This automatically:
# ✅ Updates DESCRIPTION file 
# ✅ Commits and pushes to GitHub
# ✅ Triggers GitHub Actions that:
#    - Create GitHub release v0.3.4
#    - Build Docker images (weizhou0/saigeqtl:0.3.4 + latest)
#    - Build binary packages for multiple platforms  
#    - Update all documentation automatically
#    - Upload binaries to GitHub release
```

### 3. What Happens Automatically (No Manual Work)

**Within 10-15 minutes after running `./scripts/update_version.sh`:**

1. **GitHub Release Created**: `v0.3.4` with release notes
2. **Docker Images Built**:
   - `weizhou0/saigeqtl:0.3.4`
   - `weizhou0/saigeqtl:latest` (updated)
3. **Binaries Built**: Multiple R versions + platforms automatically
4. **Users Can Install Immediately**:
   ```bash
   docker run --rm weizhou0/saigeqtl:0.3.4 step1_fitNULLGLMM_qtl.R --help
   conda install -c aryarm r-saigeqtl  
   curl -fsSL https://raw.githubusercontent.com/weizhou0/qtl/main/scripts/install_binary.R | Rscript -
   ```

## 🔧 Manual Override (If Needed)

### Manual Docker Build
```bash
# If automatic build fails
docker build -f docker/Dockerfile -t weizhou0/saigeqtl:0.3.4 .
docker push weizhou0/saigeqtl:0.3.4
```

### Manual Binary Build
```bash
# If automatic binary build fails
pixi run R CMD build .
pixi run R CMD INSTALL --build SAIGEQTL_0.3.4.tar.gz
# Move to binaries/ directory
```

### Manual Release
```bash
# If automatic release fails
git tag v0.3.4
git push origin v0.3.4
# Create release via GitHub UI
```
